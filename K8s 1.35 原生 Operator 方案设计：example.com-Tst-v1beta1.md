# 四、并发控制：锁 + 工作队列（生产最优）

## 1. 工作队列：官方 `workqueue`

**唯一正确选择**：

- `k8s.io/client-go/util/workqueue`

- 使用 `RateLimitingInterface`

- 自带：去重、限速、重试、延迟

作用：

- 同一个 Tst 对象**同一时间只进一个 worker**

- 天然防止并发 Reconcile

## 2. 资源级锁（对象级并发安全）

在队列之外，**再做一层对象级锁**，防止多 worker/多进程冲突：

- 用 `sync.Map` 做**命名空间/名称级锁**

- 对同一个 `Tst` 实例，同一时间只允许一个 goroutine 处理

最终并发安全强度：

**Leader 选举（进程级） → 工作队列（对象串行） → 对象锁（强互斥）**

三重防护，生产绝对安全。

# 六、Reconcile 核心流程设计（最优稳定）

1. 从队列获取 key（namespace/name）

2. Get Tst 对象

3. 对象不存在 → 清理逻辑

4. 对象存在 → 执行业务逻辑

5. 更新 Status（用 **Server-Side Apply 或 Patch**，1.35 推荐）

6. 异常：重新入队（带指数退避）

7. 成功：从队列确认完成

## Status 更新规范（1.35 最新）

- 使用 `Subresource` 模式：`Status()` subresource

- 用 `PatchType: types.MergePatchType` 或 `ApplyPatch`

- 不使用旧的 `Update` 全量替换（易冲突）
